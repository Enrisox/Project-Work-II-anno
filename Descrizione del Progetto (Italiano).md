# Project Work:  DevSecOps Web API & Data Analytics 

# 1. Introduzione 

Il Project Work "DevSecOps Web API & Data Analytics" mira a combinare competenze di  sviluppo sicuro, containerizzazione, gestione della sicurezza di rete e analisi dati. Gli  studenti realizzeranno un servizio web (API) completo, sviluppato in Python/Flask, e  ospitato in un container Docker, che espone funzionalità di analisi dati. Il progetto  richiederà di gestire l'esposizione sicura del servizio in un contesto simulato (rete aziendale  virtuale o Cloud), implementando hardening del server, regole di firewalling e  autenticazione per l'accesso alle API. 

# 2. Figure del team 
Si propone un team di cinque figure con responsabilità operative e gestionali specifiche. 

* **Responsabile/Project Manager (PM)** e DevOps Lead: Gestisce il progetto, definisce  obiettivi e scadenze. Ha la responsabilità della pipeline di deploy e dell'orchestrazione  del container (Docker). Assicura che le pratiche DevSecOps (sicurezza integrata nello  sviluppo) siano seguite. 
* **Viceresponsabile/Team Leader & Security e Network Administrator**: Gestisce il lavoro di  team e funge da liaison con i referenti. Operativamente, è l'esperto di sicurezza di rete e  port forwarding, responsabile della configurazione della macchina virtuale/container  server, del firewalling a livello host/rete e della corretta e sicura esposizione del servizio  (es. reverse proxy/SSL). 
* **Backend Developer & API Specialist**: L'esperto dedicato allo sviluppo dell'API REST  utilizzando Python e Flask. Responsabile della creazione degli endpoint, della gestione  delle request e dell'implementazione del meccanismo di autenticazione (es. token, basic  auth) per le chiamate alle API. 
* **Data Analyst & Python Expert**: L'esperto focalizzato sull'integrazione e implementazione  delle funzionalità di analisi dati. Lavora a stretto contatto con il Backend Developer.  Utilizza la libreria Pandas per elaborare i dati e preparare le risposte JSON/dati che  saranno fornite tramite le API.

# Obiettivi Formativi (Competenze Acquisite) 
• Acquisire competenze pratiche in sviluppo Backend (Python/Flask) e analisi dati  (Pandas). 
• Padroneggiare i principi di Containerizzazione (Docker) e Hardening di servizi web. • Consolidare la capacità di gestione della sicurezza di rete (firewall, port  management) per servizi esposti. 
• Sviluppare la capacità di prototipazione rapida (Frontend con l'ausilio di strumenti  AI). 

# 4. Fasi di realizzazione 
Il progetto sarà suddiviso in fasi sequenziali, con strumenti specifici per ogni step. 

Fase 0: Lezione Curricolare Laboratoriale – Design Architetturale  
La struttura iniziale e il design architetturale saranno sviluppati e discussi in modo  collaborativo durante le lezioni laboratoriali prima dell'avvio ufficiale del Project Work. Questo  momento didattico garantirà che tutti gli alunni della classe abbiano una visione chiara del  design e fornirà al gruppo assegnato al progetto un'importante preparazione iniziale e un  framework di partenza solido. 

# 5. Risultati attesi

Al termine del Project Work, il team dovrà presentare i seguenti risultati tangibili:
• Servizio API Funzionante: Un set di API REST (Python/Flask) con autenticazione attiva, in grado di effettuare elaborazioni dati con Pandas.
• Ambiente Server Sicuro: Il servizio ospitato su una macchina virtuale o in un container Docker (preferito), protetto da regole di firewalling e port forwarding.
• Interfaccia Web di Test: Un prototipo di frontend (HTML/JS/Bootstrap), in grado di chiamare le API e visualizzare i dati elaborati (grafici).
• Documentazione Tecnica: Manuale di Setup e Deploy, Documentazione API e Report di Sicurezza.
